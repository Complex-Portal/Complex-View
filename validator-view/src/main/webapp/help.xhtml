The validator can use two different data models to validate the files : The validator will execute a set of rules based on the PSI-MI ontology to check your file(s). It can use 4 possible scopes:
  • XML syntax
  • Usage of controlled-vocabulary defined in the PSI-MI ontology
  • MIMIx
  • IMEx
 The validator will execute a set of rules based on the PSI-PAR ontology to check your file(s). It can use 2 possible scopes:
  • XML syntax
  • Usage of controlled-vocabulary defined in the PSI-PAR ontology
The scope 'XML syntax' is not specific to the chosen data-model because both data models are using the same XML schema : PSI-XML 2.5.
The validation scopes described below define the the stringency of the validation rules applied to a dataset. As the image below illustrates, the currently supported scopes are built on top of one another. Increasing progressively their specificity.

2.5.3

This scope makes sure that the use of controlled vocabularies is correct. That is, for each controlled vocabulary place-holder, it checks that one uses the appropriate set of terms as defined in the PSI-MI ontology or in the PSI-PAR ontology depending on which data-model you first choose. The rule is currently looking at:
Interaction detection method PSI-MI PSI-PAR
Interaction types PSI-MI PSI-PAR
Participant identification methods PSI-MI PSI-PAR
Participant's experimental roles PSI-MI PSI-PAR
Participant's biological roles PSI-MI PSI-PAR
Participant's Feature detection method PSI-MI PSI-PAR
Experiment's feature detection method PSI-MI PSI-PAR
Participant's feature type PSI-MI or PSI-MOD PSI-PAR or PSI-MOD
Experiment's bibliographical references PSI-MI PSI-PAR
This scopes checks that the Minimum Information required for reporting a Molecular Interaction eXperiment (MIMIx) are respected. This scopes checks that the curation standards agreed by the International Molecular Exchange Consortium (IMEx) are respected.
