Experiment, Interaction, Interactor? Please explain?

An experiment consists of many interactions which contain two or more interactors. An interactor can be either an individual protein or a protein complex (i.e. the result of a previous interaction). Therefore an interaction can consists of two or more protein or complex. Let me explain:

In order to model molecular interactions IntAct has designed three (virtual) objects; Experiment, Interaction and Interactor. For example the two interactions (A & B) can be described by a single Experiment. InteractionA consists of two interactors protein1 and protein2, the two interactors in InteractionB are InteractionA and protein3. Now read the first paragraph again and see if it makes more sense.




More Details

Each Object (Experiment, Interaction, Interactor) has attributes assigned which provide a detailed description. This is important as specific features of an experiment can have a profound impact on the type of interaction. For example:

  •   Host organism - experiments with the same gene from the same organisms might behave differently when studied in a different host, for example proteins expressed in a yeast 2-hybrid system loose all temporal and spatial control of their expression pattern imposed in their normal cellular environment. Protein expression system – eukaryotic proteins may loose post translational modifications essential for molecular interactions when expressed in a bacterial system.

Accession Number

IntAct Accession. Every IntAct object has a unique accession code which starts with "EBI-" and then a number. It is these accession codes that enable the hierarchical data structure.

Description

A brief statement describing the contents of the object




Experiment
Attribute
Description
Interaction identification

The experimental method by which the interaction has been detected.
Example: yeast two hybrid.

Participant identification

The experimental method by which the molecules participating in the interaction have been detected. Example: Mass spectrometry.

Host

The biological system in which the interaction took place in the experiment. Example: Yeast ( Saccharomyces cerevisiae ); the host organism of a yeast two hybrid experiment is "Yeast" even when the system is being used to investigate human proteins.

Annotation

Comment for public view

Cross reference

Cross-references to other databases. A cross-reference has the following attributes:

  • Name:The name of the external database.
  • Primary id: The primary id or accession number of the object in the external database.
  • Secondary id: The secondary id of the object in the external database.
  • Crossreference type: The type of crossreference.

Interactor (Protein)
Attribute
Description
Source

The biological system in which the interactor normally is found. For a human protein, the BioSource would be "Human", even if it has been expressed in Yeast in a particular experiment.

Gene names

Gene names.

Cross reference

Cross-references to other databases with information on the specific protein.

Sequence length

Number of amino acid sequences which constitute the protein

CRC64 checksum

The CRC64 checksum for the amino acid sequence. This checksum is used to quickly detect if the amino acid sequence has changed, for example during an update process. It is also used to check if the amino acid sequence has potentially been corrupted. The algorithm to compute the CRC64 is described in the ISO 3309 standard. The generator polynomial is x64 + x4 + x3 + x + 1. Reference: Press W.H., Flannery B.P., Teukolsky S.A. and Vetterling W.T. "Numerical recipes in C", 2nd ed., pp896-902, Cambridge University Press (1993). It should be noted that while, in theory, two different sequences could have the same CRC64 value, the likelihood that this would happen is extremely low.

Sequence

Amino acid sequence of the protein
