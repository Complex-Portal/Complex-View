Intact Search Application

    1) Basic Overview

    The search application currently allows a text-based search by entering a text string into the search text box. It is possible to obtain information on experiments, protein interactions and the proteins themselves, with various views used to display the data. Intact uses the concept of short label which is intended to provide a way of 'more memorable identification' compared to a standard Accession Number (AC), although searches by AC are obviously allowed. For example a particular Experiment may have an AC of EBI-14, but its short label is 'gavin' indicating a more meaningful reference to the physical experiment which the entry represents. Basic points on usage:
    • Wildcard searching is supported - for example matches will be retrieved for strings such as EBI*, *BI-14, E*4, etc etc.
    • upper or lower case strings are acceptable and will be matched if they exist
    • currently searches progress in a certain order: proteins, then interactions, then experiments. At a lower level, specific attributes are searched by AC, short label, Xref primary ID and finally full name. This algorithm may change as the intact project matures.
    • Note that whilst CvObject details can be displayed via the click of a link, currently it is not possible to search for CvObjects via the text box search entry point.

Definition of Table Items

The items displayed in the above table format have the following descriptions:

AC - The Accession Number of the Experiment, Interaction etc.

ShortLabel - a 'more memorable' way of identifying the Experiment, Interaction etc other than by its AC. A short label should be unique within particular classes of objects (for example within CvTopics) but in general they may not be.

full name - the complete name of the Interaction, Protein etc (eg ????)

CvInteraction - a link to a page displaying the controlled vocabulary information for the term 'Interaction'

CvIdentification - a link to the controlled vocabulary information for the means of identifying an item (eg ???)

BioSource - the biological source of the interaction information (eg the organism it was observed in)

Comment - annotation text, usually split into categories of annotation topic,  describing further details of the interaction, experiment etc.

xref database - a link to the information describing a cross-reference in another database (eg swissprot, GO etc)

primaryId - the original source of information

secondaryId - ???

DbRelease - the version of the cross-reference database that the data relates to

There is also:
A row listing the short labels of the proteins involved in a particular interaction - each short label has a link to the controlled vocabulary describing the role that the protein plays in this interaction (see below). Each short label is a link to more detailed data on a particular protein.

In addition to these there are a number of other items which may be displayed in various views, particularly with respect to interaction data:

CvComponentRole - a link to the controlled vocabulary information describing the role that an interactor plays in the interaction (eg for a protein it may have a role of 'bait' or 'prey' in a 2-hybrid experiment)

CvInteractionType - a link to the controlled vocabulary information describing the type of interaction observed (eg ???)

Stoichiometry - a numberical value attributed to a component of an interaction, signifying ?????

Protein short labels are links you can click to get a view of the protein details. Also each protein short label has a term in brackets (eg '[p]') which denotes the role that the protein plays in the interaction. These terms have tooltips associated with them to display the role name, and also the term is a clickable link from which you can view the full details of the controlled vocabulary term which presents the controlled vocabulary information for that term (eg the details held to define the meaning of the term 'bait').

Since the items displayed for a search result include many items not actually queried for, the items that matched the oringinal search string are highlighted in bold italic - there is even a helpful little message at the top of the results page to remind you of this. Specifically the *short labels* of the matching items are highlighted, since these are always displayed in every view.

Where a protein (for example) might actually occur more than once in an experiment, the display is rationalised to avoid multiple presentations of the same experiment - therefore all occurrences within an experiment are gathered together into one experiment view.

At the bottom of the page are a number of buttons:

    • a button to view the full details of what is on the results page (this expands all the information currently on the page, apart from the controlled vocabulary details);
    • a reset button to uncheck any textboxes;
    • a graph button to display a graph of any selected proteins, which can be chosen by clicking the appropriate checkboxes.
